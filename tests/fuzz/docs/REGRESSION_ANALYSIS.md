# Regression Analysis: 96% → 80% → 88% Pass Rate Journey

## Timeline

1. **Before Tailwind v4 sync**: ~96-100% pass rate with 341 custom properties
2. **After Tailwind v4 sync** (commit 3758006): 80% pass rate with 337 properties
3. **After adding 4 properties** (first fix): 90% pass rate with 341 properties
4. **After adding 4 more properties** (second fix): 88% pass rate with 345 properties

## Root Cause Analysis

When syncing with Tailwind CSS v4's canonical property order (commit 3758006), **8 properties** were removed that were critical for Prettier compatibility:

### Removed Properties (causing 16 percentage point drop)

1. **`background-opacity`** (was at index 0)
   - Used by: `bg-opacity-*` utilities
   - Status: ✅ **Restored** at index 0

2. **`border-opacity`** (was at index 177)
   - Used by: `border-opacity-*`, `divide-opacity-*` utilities
   - Status: ✅ **Restored** at index 182

3. **`--tw-prose-component`** (was at index 262)
   - Used by: `prose`, `prose-sm`, `prose-base`, `prose-lg`, `prose-xl`, `prose-2xl`
   - Status: ✅ **Restored** at index 267

4. **`--tw-prose-invert`** (was at index 263)
   - Used by: `prose-invert`
   - Status: ✅ **Restored** at index 268

5. **`outline-style`** (was at index 335)
   - Used by: `outline-solid`, `outline-dashed`, `outline-dotted`, `outline-double`
   - Status: ✅ **Restored** at index 314

6. **`--tw-divide-x-reverse`** (was at index 337)
   - Used by: `divide-x-reverse`
   - Status: ✅ **Restored** at index 126

7. **`--tw-ring-inset`** (was at index 304)
   - Used by: `ring-inset`
   - Status: ✅ **Restored** at index 309

8. **`user-select`** (was at index 336)
   - Used by: `select-none`, `select-text`, `select-all`, `select-auto`
   - Status: ✅ **Restored** at index 344

## Why These Properties Were Missing from Tailwind v4

Tailwind CSS v4's `property-order.ts` (337 properties) does NOT include these properties because:

1. **Tailwind v3 backwards compatibility**: `background-opacity`, `border-opacity` are v3 properties
2. **Synthetic properties**: Some properties like `user-select` aren't generated by Tailwind v4 CSS but are used for sorting
3. **Experimental features**: `--tw-prose-*` properties for typography plugin
4. **Runtime-only properties**: Properties used for sorting but not in final CSS output

## Impact of Restoring Properties

|-------|-----------|-----------|-------------|
| Tailwind v4 sync | 337 | 80% | Baseline (regression) |
| Added 4 properties (first fix) | 341 | 90% | +10 percentage points |

*The second fix actually improved some cases but may have shifted indices causing other issues.

## Current Status (345 properties)

**Pass Rate**: ~88% (87-91% range across tests)
**Improvement from regression**: +8 percentage points
**Gap to original**: -8 percentage points (96% - 88%)

All 8 identified missing properties have been restored, but pass rate is still slightly below the original 96%. This suggests:

1. **Index shifts**: Adding properties at different positions may have changed sorting behavior
2. **Additional missing properties**: There may be other properties not yet identified
3. **Variant ordering**: Issues may be in variant_order.rs, not property_order.rs
4. **Algorithm differences**: Subtle differences in property counting or tie-breaking
